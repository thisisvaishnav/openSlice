# Mission: Win the SigNoz OpenTelemetry Hackathon

## Why
Prepare for the July 20-26, 2026 SigNoz hackathon by learning enough OpenTelemetry, SigNoz, and maintainer-grade open source practice to build a standout project and become a serious future contributor to OpenTelemetry Collector Contrib.

## Success looks like
- Explain the hackathon tracks, judging criteria, and hidden scoring signals clearly.
- Choose a project idea that uses SigNoz deeply: traces, metrics, logs, dashboards, alerts, Query Builder, MCP, and reproducible Foundry deployment.
- Build a demo that tells a real debugging story, not just a feature checklist.
- Make credible, respectful contributions to OpenTelemetry Collector Contrib before and after the hackathon.

## Constraints
- This is the user's first hackathon.
- Coding/design work for the hackathon itself should begin only after the hackathon starts; notes, sketches, diagrams, and strategy are allowed beforehand.
- The user is highly motivated and willing to work hard, but currently has only early-stage OpenTelemetry understanding.
- AI assistant use must be disclosed in the submission.

## Out of scope
- Building hackathon code before the official start.
- Chasing every OpenTelemetry subsystem at once.
- Optimizing for flashy demos that do not use SigNoz deeply.
