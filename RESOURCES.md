# SigNoz Hackathon and OpenTelemetry Resources

## Knowledge

- [Agents of SigNoz Hackathon Page — WeMakeDevs](https://www.wemakedevs.org/hackathons/signoz)
  Use for: official dates, tracks, prizes, judging criteria, FAQ, blog prize, and submission expectations.
- [SigNoz Docs: Install SigNoz on Docker Standalone](https://signoz.io/docs/install/docker/)
  Use for: the official Foundry-based Docker install flow, `casting.yaml`, `casting.yaml.lock`, ports, and enabling the MCP server.
- [Foundry Docs: MCP Server](https://github.com/SigNoz/foundry/blob/main/docs/concepts/mcp-server.md)
  Use for: understanding how Foundry deploys the SigNoz MCP server, authentication modes, the `/mcp` endpoint, and AI-client setup.
- [SigNoz Docs: Introduction](https://signoz.io/docs/introduction/)
  Use for: the high-level SigNoz product map and official navigation into docs for traces, logs, metrics, dashboards, alerts, and APIs.
- [SigNoz Docs: Instrument Your Application](https://signoz.io/docs/instrumentation/)
  Use for: language-specific OpenTelemetry setup when selecting the hackathon stack.
- [SigNoz Docs: Query Builder v5](https://signoz.io/docs/userguide/query-builder-v5/)
  Use for: building judging-visible dashboards, trace/log/metric exploration, and alert queries without raw SQL or PromQL.
- [SigNoz Docs: Manage Dashboards](https://signoz.io/docs/userguide/manage-dashboards/)
  Use for: creating polished visual evidence for the demo and submission.
- [OpenTelemetry Docs: Collector](https://opentelemetry.io/docs/collector/)
  Use for: understanding receivers, processors, exporters, pipelines, and Collector deployment patterns.
- [OpenTelemetry Demo](https://opentelemetry.io/docs/demo/)
  Use for: learning how realistic multi-service telemetry is generated and visualized.
- [OpenTelemetry Collector Contrib: CONTRIBUTING.md](https://github.com/open-telemetry/opentelemetry-collector-contrib/blob/main/CONTRIBUTING.md)
  Use for: PR titles, changelog rules, local testing, AI-assisted contribution rules, issue triage, and maintainership expectations.
- [OpenTelemetry Collector Contrib Repository](https://github.com/open-telemetry/opentelemetry-collector-contrib)
  Use for: reading real components, metadata files, tests, CODEOWNERS, issues, and active PR style.
- [OpenTelemetry Semantic Conventions: GenAI directory](https://github.com/open-telemetry/semantic-conventions/tree/main/docs/gen-ai)
  Use for: AI-agent observability naming, GenAI spans, model/tool attributes, and MCP-related telemetry.
