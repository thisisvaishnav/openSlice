Lesson 0002

Today's Prep Sprint

July 14, 2026 is not a build day. It is a leverage day: install, observe, write, and sharpen the winning story.

Today's target: End the day with SigNoz running locally through Foundry, one toy telemetry flow explored, and a pre-event blog outline ready to draft.

Block 1: Understand the battlefield

Timebox: 45 minutes. Read the official hackathon page once with a notebook open. Extract only four things: tracks, judging criteria, required submission artifacts, and pre-event blog rules.

Deliverable: Write a 6-line judging matrix: Impact, Creativity, Technical Excellence, Best Use of SigNoz, UX, Presentation.

Block 2: Install SigNoz the judge's way

Timebox: 90 minutes. Follow the official Docker install guide using Foundry. Your goal is not mastery yet; it is to learn the reproducible path and produce the files judges expect later.

Deliverable: A short note answering: What does casting.yaml describe? When is casting.yaml.lock created? Which ports matter?

Block 3: Send toy telemetry

Timebox: 90 minutes. Use a tiny practice app or example, not the final hackathon idea. Send one trace, one log, and one metric into SigNoz if possible.

Deliverable: Three screenshots or notes: where the trace appears, where logs appear, and where a metric/dashboard panel appears.

Block 4: MCP reconnaissance

Timebox: 45 minutes. Read the Foundry MCP server doc. Do not overbuild. Understand the endpoint, authentication modes, and why MCP helps an AI assistant query observability data.

Deliverable: One paragraph: "In my hackathon demo, MCP could let an agent..."

Block 5: Maintainer visibility habit

Timebox: 60 minutes. Read five OpenTelemetry Collector Contrib issues and two merged PRs in one component area. Do not comment yet unless you can add real value.

Deliverable: List one component that feels approachable and one contribution style you can attempt later: docs, tests, bug reproduction, or small code fix.

Block 6: Pre-event blog outline

Timebox: 45 minutes. Draft the shape of a proper blog for the July 19 pre-event prize. Make it honest and concrete.

Suggested title First Contact with SigNoz: Making My Toy App Observable Before the Hackathon

Deliverable: Outline five sections: why observability matters for AI agents, setup, first telemetry, what confused me, what I will build during the hackathon.
