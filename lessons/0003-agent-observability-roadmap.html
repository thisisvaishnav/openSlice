Lesson 0003

Build Agents with End-to-End Observability

The beginner path for building an AI agent project where SigNoz is central to debugging, not decoration.

Target track: Observability for AI / Agent systems. Build an agent, make it fail in realistic ways, and use SigNoz to explain every important step from user request to root cause.

The mental model

An AI agent is a loop: receive a task, reason, call tools, observe results, call an LLM, maybe retrieve context, and return an answer. End-to-end observability means every step in that loop emits useful telemetry.

TracesShow the journey: user request, LLM call, tool call, database/API call, and final answer.
LogsExplain events: selected tool, retry, failure, warning, prompt summary, or validation result.
MetricsMeasure patterns: latency, error rate, token usage, cost, retries, and success rate.
DashboardsTurn raw signals into a judge-readable investigation view.
AlertsDetect bad behavior: high latency, too many errors, or runaway token usage.
MCPLet an AI investigator query SigNoz and explain failures with evidence.

What to study first

  1. OpenTelemetry basics: service name, traces, spans, attributes, logs, metrics, OTLP.
  2. SigNoz basics: where traces, logs, metrics, dashboards, alerts, and Query Builder live.
  3. Agent basics: LLM call, tool call, retrieval, retry, error, and final response.
  4. Instrumentation: how code emits spans, logs, metrics, and useful attributes.
  5. Demo storytelling: create a failure, inspect evidence, explain root cause.

The final project shape

Build a small agent that performs a real task, such as checking a fake support ticket, querying a mock service, or investigating a toy incident. Then instrument the full path. The winning demo is not "look, an agent." The winning demo is "look, this agent failed, and SigNoz shows exactly why."

Pre-hackathon practice

Before July 20, practice with toy scripts only. You can learn how to send telemetry, create dashboards, and write the blog. Save final project coding for the official build window.

Retrieval drill

What makes this project end-to-end observable?

Ask the agent to quiz you on traces, spans, logs, metrics, and agent tool calls before you build the final version.
