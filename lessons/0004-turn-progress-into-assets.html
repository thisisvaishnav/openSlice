Lesson 0004

Turn Progress Into Assets

July 15 is about making yesterday's work usable: screenshots, blog draft, dashboard practice, and a sharper agent-observability design.

Today's target: Build an evidence pack and draft the pre-event blog. The prize is not just that SigNoz runs; it is that you can explain what you learned clearly.

Block 1: Evidence pack

Timebox: 45 minutes. Re-run the telemetry loop and capture clean screenshots. Name each screenshot by what it proves.

  • 01-docker-containers-running
  • 02-signoz-service-hackathon-toy-app
  • 03-trace-first-smoke-test
  • 04-log-first-telemetry-message
  • 05-metric-latency-visible

Block 2: Dashboard practice

Timebox: 60 minutes. Create one dashboard panel for hackathon.toy_agent.latency_ms. Make the panel title human-readable: "Toy Agent Latency".

Deliverable: One screenshot of the dashboard and one sentence explaining what the panel shows.

Block 3: Blog draft

Timebox: 90 minutes. Draft the pre-event blog. Keep it honest and beginner-friendly. Write what happened, not what sounds impressive.

Blog skeleton Problem: AI systems are hard to debug. Setup: I installed SigNoz locally. First signal: I sent trace/log/metric data. Confusion: what was unclear. Learning: how OTLP and ports fit together. Next: agent observability during the hackathon.

Block 4: Agent observability design

Timebox: 60 minutes. Sketch the final project without coding it. Draw the agent loop and label where traces, logs, metrics, dashboards, alerts, and MCP appear.

Deliverable: A one-page design note titled "Agent Observability Demo Story".

Block 5: Retrieval drill

What changes today from yesterday?

Bring the blog draft back to the agent for review. Ask for ruthless feedback on clarity, evidence, and whether it sounds like a real learner rather than a marketing post.
